# Protocol discussion
- Protocol must be presented at the latest 2 days prior experiment to PI.
- Staining strategy should be presented at project meeting with PI.
- Protocol must be sent to PI with revisions if applicable for final approval.
# Experiment execution and documentation during an experiment
- Check availability of all reagents needed for the experiment, equipment and members of staff who may be involved in the procedure.
- You must have been approved by the person responsible of the technique to undertake the technique autonomously.
- You must not deviate for the standard protocol unless agreed by PI and the person responsible for the technique.
- If such deviations are made, ensure that ALL details are clearly stated in the lab book.
- Document the protocol lay out onto the lab book with all additional information like calculations, change of antibody lot number, location of cytospins, file names of FACS plots, qPCR data…
- Test experiment, titrations, batch testes of reagents should be stored as specific project in KI-ELN rather than a specific experimental Project.
- To ensure optimal and timely use of ELN, new lab members will first be informed in detail by lab managers and the PI, and then lab book entries checked and discussed at least 1-2 times (and if needed more) with one of the lab managers, the first 2-3 months, then every 6 months.
- The Lab Managers have the delegated responsibility to ensure/check that the rules in this document and the handling of ELNs are followed carefully by all lab members.
# Analysis/Summary of data
- Analysis must be completed as soon as possible (~1 week) after the experiment.
- If any data is modified for the analysis, this must be clearly stated and justified in the lab book.
- Data should be compiled and easily linked to raw data should inspection of the initial data be needed. Use the same rules for naming files, eg; Project_experiment number_ date the experiment started_name of experiment.
- Example Flt3_001_20160426_TAMtest.
- Analysis will always be double checked by a second lab member before a paper can be submitted.
# Storage of Data
- All the data, except analysis files and electronic files should be kept in the ELN lab book. It should be clearly indicated where all the electronic data on the project are kept. 
- **FACS data:** Use “Project000_yyyymmdd_Initials” to name files, refer to the file name as appropriate in the lab book to ease search for data. Store raw data at portable disc and back up in own KI-Box.
- **Sequencing data:** On OneDrive, hard drive and ELN.
- **Q-PCR data:** Electronic data is kept by the responsible person, cross reference the folder where the data can be found. Store raw data at portable disc and back up in own KI-Box.
- Morphology data:** Cytospin slides must be kept in a labeled slide box. The location of which should be made clear in the lab book. Slides and files should be named according to the standard format: project name_exp number_date_initials.
- Sequencing data:** Stored on Fuji, see separate SOP

# Checking of data
- The responsible person for the project must ensure that data is double checked.
- The responsible person for the project must make double checking as easy and straight forward as possible.
- Analysis will always be double checked by a second lab member before a paper can be submitted.
- If raw data is inaccessible, then the analysis cannot be published.
- Double-checking of data by a second investigator/mentor, with expertise in involved techniques is strongly encouraged, and if this is done information about this and potentially co-signing should be done. Cosigning of an experimental protocol or data should never be done unless the protocol/data have been carefully checked by the second person.
- At the end of a project (time of potential publication), all data and corresponding entries in the ELN are double-checked, first by the project responsible and then by at least a second person with expertise in the involved techniques. This is done to ensure that we have not made any human errors when conducting experiments, in documentation of data (according to these and international guidelines), that data are properly stored and accessible, that calculations and presentation of data are correct and state-of-the-art, and that no relevant data have been excluded unless properly justified. This careful double-checking and documentation of data before a publication, is included in a folder including all the figures/tables in the submitted manuscript, with the raw data and calculations, comments etc attached, to be kept in the lab by Lab Managers. If experimental protocols and data have not been properly documented and stored, we cannot according to international guidelines publish them. After publication the folder will be duplicated and one copy stored in the lab and one copy archived by the personal assistant. The folder will be kept for 10 years.
- **The checking of data involves:**
	- **Lab book** (protocol, calculations, proper cross-reference between data)
	- **FACS data** (correct gates have been set, all the required info included, proper cross-references have been done between Lab book and FACS data, values in table corresponds to correct values in FACS profiles).
	- **Figures:** Make a binder with Figure/tables and attached raw data for all submitted manuscripts, specify if experiments have been left out alongside its justification.
	- **Data in main text:** Checking that data that is referred to in the main text is correct.
	- Each paper data will be gathered in a binder which will be kept in the lab.
	- When all the data has been checked, at least two persons should have signed with name and date. Only then is the data fulfilling the requirements for publication.
- **NB: A manuscript cannot be submitted if not ALL the data has been checked by at least two people.**
# Documentation of data for publication
- Performed by lab managers when paper is ready for submission. Main objective is to make sure all data included in submission is documented for anyone to be able to find.
- All raw data and analysis should be stored in an external hard drive “Experiment_X”. The data should be organized into folders according to the main header of each figure.
- All the original protocols for Experiment_X should be located in the same folder.
- All protocols from Oxford should be in the log book and clearly marked with stickers at the top or side.
- Clearly state experimental number, patient numbers, summary of treatment, mice genotype used by each figure
- Documentation also include genotyping of mice, experimental logbook from animal facility, raw data files (FACS, Fluidigm, Sequencing etc)
- Reason for exclusion of data, mice, time points, samples must be stated
# ELN Guidelines
- Logbooks must be maintained, and entries made for all experiments, observations and/or measurements. Any alteration must be dated and signed and entered in such a way that the original text is still legible.
- Experimental records must be kept with dates and entries by the person responsible for the conduct of the experiment. The experimental record must be formulated in such a way that the experiments can be repeated, even after an interval of several years, and in another laboratory. This requires documentation of both the experimental material (animals, cells etc) and equipment and measurement procedures, chemicals, isotopes etc., which are of importance to the results.
- As soon as possible after completion, reports of the experiment must be compiled. The report must contain or have as appendices information on the calculations and statistical analyses, and a description and justification of any alterations, so that it is possible to reconstruct how the results were arrived at. All research documents must be written in English and be intelligible to an outsider.
- Information about quality control of the accuracy of measurements, and the methods and computer programs used for analysis of results should also be available. It should be possible to assess experimental variation within and between experiments, reproducibility and possible errors of methods.
- As far as possible the results should be stored in the form of raw data, in legible and easy identifiable form e.g. permanent printouts, of electronic data, printouts from measurement instruments, lasting photographs of morphological data or molecular biological analyses (e.g. gels) which can be identified with certainty, printouts of statistical calculations etc. Even unpublished results should be kept. From the archives, it must be possible to identify the original observations on which published tables and figures were based (e.g. by referring to the protocol number of the relevant experiments.)
- The project leader must retain on file approval granted by the various authorities (e.g. from The Local Ethics Committee on Animal Experiments). As soon as possible after completion of the project, the original data, protocols etc. must be prepared for archiving; if the data are to be used for subsequent projects, it must be stated clearly how and where they are being stored. The results must be archived for a reasonable time, retrievable for review or critical enquiry, i.e. at least 10 years after publication of the results. Please see KI’s current archiving plan for further details on saving and discarding of records, as certain records must be preserved for more than 10 years. Where applicable, the need for confidentiality (e.g. where there are commercial interests) must be respected, both during conduct of experiments, and with respect to archiving.
- The Department (Head of Department) is responsible for the archives; scientists should have access to copies of data from their own projects. If the scientist in charge of the projects leaves KI, an agreement with respect to access to the archived data base must be made with the responsible head of department.
- Add documents to ELN as original Word files, Excel files, PDF:s as much as possible. Do not use option “File attachment” if file can be added as an original file.
- Experiment title should include project_experimentnumber_ date the experiment started_name of experiment. Example: Flt3_001_20160426_TAMtest
- All files included in the experiment should be named according to the same rules.
- **Start experiment** **with a “body text”** containing a summary of experiment parts with dates (if necessary) and location of stored data. This section is left unlocked until submission
- Reference to where data is stored should always be included e.g. location of slides, where the raw data is stored (e.g. FACS, microscope slides etc) and the name of the file.
- Include “related experiment section” if relevant (genotyping data, other timepoints to same experiment etc)
- The Jacobsen lab has a number of SOP protocols for specific methods, which should be included in the ELN if used. It is important that the exact version of the SOP used is included as this regularly updated/changed. When repeating the use of a SOP, one can refer to the previous entry if the SOP has not changed, but if an updated version is used this must be entered
- Changes to experimental protocols after these were originally added to the lab book or changes to an SOP should be made in red and always include a date for the entry. If a part of a protocol was not performed, e.g. not enough cells or an infection occurred, write a comment in red why the part was excluded from the experiment. The part excluded could also be marked in a different colour, e.g. yellow.
- Sufficient details must be provided in protocols about used reagents, such as Antibody concentration, clone name and batch number, batch number for serum etc.
- During FACS analysis reference between antigen and conjugate **MUST** be made in Experiment - Experiment layout. Write each antigen in the column for the corresponding conjugate.
